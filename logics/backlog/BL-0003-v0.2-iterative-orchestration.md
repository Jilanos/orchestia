## Backlog Slice

Define v0.2 as the first iterative need-to-completion orchestration loop.

## Scope

- Initial need intake.
- Primary need decomposition.
- Request creation per primary need.
- Backlog and task generation.
- Codex prompt generation and execution.
- Diff, logs, and test collection.
- Review decisions using accept, revise, split, or reject.
- Looping until completion, explicit out-of-scope decision, or firm blocker.
- Controlled auto branch execution, auto push, and auto merge policy for fresh projects or isolated branches.

## Done When

- The v0.2 macro and micro loops are documented.
- Firm blocker criteria are documented.
- Human control points are explicit.
- Controlled auto push and merge guardrails are documented.
- No automation has been implemented before policy and template updates.
