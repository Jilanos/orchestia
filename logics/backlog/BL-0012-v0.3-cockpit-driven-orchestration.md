# BL-0012: v0.3 Cockpit-Driven Orchestration

## Metadata

- ID: BL-0012
- Status: Active
- Related release goal: v0.3 Cockpit-driven orchestration

## Objective

Move Orchestia from a cockpit that prepares and monitors orchestration to a cockpit that can drive the orchestration workflow with explicit human confirmations and preserved safety boundaries.

## Backlog Scope

- Need intake to Logics draft generation.
- Task prompt draft preparation.
- Explicitly confirmed autonomous-loop launch.
- Refreshed loop status and evidence views.
- Token usage evidence improvements.
- Review finalization with explicit decisions.
- Controlled Git flow handoff preparation.
- v0.3 negative-path validation and release readiness.

## Candidate Tasks

- Cockpit action: convert need intake into Logics draft.
- Cockpit action: prepare task prompt draft.
- Cockpit action: launch autonomous-loop with explicit confirmation.
- Cockpit view: live or refreshed loop run status.
- Cockpit view: token usage improvements.
- Cockpit action: finalize reviews from UI with explicit decision.
- Cockpit action: prepare controlled Git flow handoff.
- v0.3 negative-path validation suite.
- v0.3-alpha release readiness.

## Acceptance Criteria

- The roadmap and design docs are coherent and reviewed.
- No v0.3 implementation is attempted in this planning task.
- Browser actions remain explicitly confirmed and bounded.
- Push, merge, and controlled Git flow execute remain guarded.
- Human control over blockers, ambiguous next needs, and high-risk actions is preserved.

## Risks

- UI execution controls can make risky actions feel too easy.
- Draft-to-final Logics promotion needs strict collision and review handling.
- Live status can mislead if timestamps and refresh semantics are unclear.
- Token usage display can overstate precision if missing values are not explicit.
