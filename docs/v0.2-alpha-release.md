# Orchestia v0.2-alpha Release

## Release

- Release name: `v0.2-alpha`
- Release date: 2026-05-23
- Reviewed Orchestia commit: `835d13e Validate full controlled chain`
- Release decision: accept

Orchestia `v0.2-alpha` is not production-ready. It remains a human-supervised local orchestration system with explicit execution, review, push, merge, and advancement controls.

## Summary

`v0.2-alpha` consolidates the first validated state-driven orchestration chain:

- Loop state can point to a prepared Codex prompt.
- `auto-loop` can execute Codex through `codex exec --sandbox workspace-write` when explicitly authorized with `--execute-codex`.
- Execution evidence is captured under `task-runs/`.
- Review drafts can be finalized into versioned Logics reviews with explicit decisions.
- Controlled Git flow can push an isolated branch and merge it into an authorized non-protected branch.
- The local cockpit can inspect Loop state, auto-loop evidence, reports, errors, reviews, and human action required states.

## What Is Supported

- Dry-run-first loop inspection and command preview.
- Explicit Codex execution through `--execute-codex`.
- Evidence capture for Codex stdout, stderr, exit code, workspace status, diff stat, recent commits, and tests.
- Human-provided review decisions: `accept`, `revise`, `split`, or `reject`.
- Final review creation through `finalize-review`.
- Controlled auto-push and controlled auto-merge through `scripts/controlled_git_flow.sh`.
- Local read-only cockpit inspection through `scripts/orchestia_ui.py`.

## What Is Still Manual

- Choosing the next primary need when ambiguous.
- Declaring firm blockers.
- Deciding accept, revise, split, or reject.
- Authorizing Codex execution.
- Authorizing controlled Git push or merge execution.
- Advancing Loop state with explicit fields.
- Inspecting evidence before using the workflow on higher-risk projects.

The cockpit is read-only in this release. It does not execute Codex, push, merge, update Loop state, create reviews, or expose environment variables.

## Known Limitations

- This is an alpha release and not production-ready.
- Negative-path validation remains incomplete for protected branch refusal, failed-test blocking, dirty workspace refusal, and merge conflict behavior.
- Loop state advancement remains explicit and was not automated in the full-chain validation.
- The sample project is small and does not represent complex CI, deployment, or multi-repository workflows.
- GitHub repository creation was not automated in Codex for this environment.

## Risks

- `workspace-write` allows edits inside the verified target workspace and must remain behind explicit authorization.
- Generated evidence and review drafts still require human review.
- Controlled push and merge are powerful operations even with guardrails.
- The local cockpit is intended for localhost use only.

## Non-Goals

- Full autonomy.
- Background agents or daemons.
- Production deployment automation.
- Secret management automation.
- Unrestricted push or merge.
- Force push, branch deletion, rebase, or tagging by automation.
- Merge into protected `main` or `master` without explicit human-approved policy.
