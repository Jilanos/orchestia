Validate that Orchestia `v0.2-alpha` guardrails refuse unsafe, incomplete, or blocked flows without using GitHub remotes, modifying the sample todo CLI project, or running Codex against real projects.

## Scope

- Validation date: 2026-05-24
- Orchestia commit under validation: `b55b9d0`

- No GitHub remotes were used.
- No sample todo CLI files were modified.
- No Codex execution occurred against Orchestia or the sample todo CLI project.
- Disposable remotes were local bare Git repositories under `task-runs/`.
- Protected branch and protected target operations were refused without override.

## Blockers

None.

## Known Limitations

- The suite validates local script guardrails with disposable repositories; it does not exercise GitHub network behavior.
- The merge conflict case validates local Git conflict handling, not hosted pull request or CI behavior.
- The suite intentionally avoids real Codex execution and validates missing prompt, dry-run, and dirty workspace gates instead.
